Was sind Anwendungsfalldiagramme?

  • auch Use Case Diagramm genannt
  • beschreibt das externe Systemverhalten aus Anwendersicht
  • stellen das geplante System, die Akteure, die Verwendung des geplanten Systems und die Beziehungen zwischen Akteuren und Anwendungsfällen dar
  • geben Auskunft, was ein geplantes System aus sicht des Benutzers leisten soll

Das sind die Elemente von Use Case Diagrammen

System Use Case (Anwendungsfall)
Akteur Assoziation
include-Assoziation extend-Assoziation
Extension point Beispiel für include + extend

Bedeutung des Use Case Diagramms

  • hilft bei Festlegung von Anforderungen an ein Softwaresystem
  • dient als gutes Kommunikationsmittel zwischen Systemanalytiker, Anwender und Entwickler
  • zeigt Funktionalität aus Sicht des zukünftigen Nutzers betrachtet (für Analyse betriebsintern bzw. mit dem Kunden)
  • Implementierungsschritte müssen nicht beachtet werden

Zusammenhang

  • Use Cases stellen eine Menge von Systemfunktionen dar
  • diese müssen während des Projektes detaillierter beschrieben werden
  • möglich durch andere Diagramme oder zusammenhängende Texte
  • aus Spezifikationen können Klassenkandidaten abgeleitet werden

Mögliche Herangehensweise beim Erstellen

  1. (Besitz eines "Vorprogrammes", das geändert werden soll)
  2. Überlegung welcher Vorgang wird betrachtet
  3. Überlegung der Teilnehmer
  4. Überlegung welche Möglichkeiten stellt das System
  5. Verknüpfung Teilnehmer mit Möglichkeiten
